# ADR-0004: [努力値・個体値vs能力ポイント]

## ステータス

**承認**

| 日付       | ステータス | 備考 |
| ---------- | ---------- | ---- |
| 2026-03-29 | 提案       |      |
| 2026-03-29 | 承認       |      |

## コンテキスト

Pokemon Championsでは個体値・努力値がなくなり能力ポイントになった。能力ポイントを1上げると実数値が1上がる。これはLv.50固定・個体値31固定にした際の努力値を簡略化したものである。また、性格は「能力補正（Stat Alignment）」に名称変更された（仕組みは従来の25種と同じ）。
今回のPokemon Championsの仕様に対応するか、過去の形式にも対応するか。

## 検討した選択肢

### 選択肢1: [能力ポイントのみ採用]

- 概要: 個体値・努力値をなくし能力ポイントのみにする
- メリット: Pokemon Championsに対応
- デメリット: 過去の形式に非対応

### 選択肢2: [個体値・努力値のみ採用]

- 概要: 能力ポイントを採用せず、個体値・努力値のみにする
- メリット: 過去の形式に対応
- デメリット: Pokemon Championsに非対応

### 選択肢3: [個体値・努力値も能力ポイントも採用]

- 概要: 個体値・努力値も能力ポイントも採用
- メリット: 過去の形式にもPokemon Championsにも対応
- デメリット: 旧形式と能力ポイントの間で整合が取れない。以下に導出を示す。
  1. Pokemon Championsでの能力ポイント合計上限は66
  2. 旧形式（Lv.50・個体値31固定）で能力ポイントに相当する実数値上昇を再現するには、努力値合計508+8=516が必要になる
  3. しかし旧形式の努力値合計上限は510であるため6ポイント不足する
  4. 不足分8を個体値31の別パラメータに4ずつ振ると各ステータスの実数値が+1され、合計で実数値を67上げることが可能になる
  5. これは能力ポイント合計66と一致せず、両形式の互換性を保てない

## 決定

[選択肢1] を採用する。

理由: 対戦環境の場はPokemon Championsに移りつつあり、本ツールの主要ターゲットである対戦補助機能はChampionsの仕様に完全対応すべきである。また能力ポイント制の採用により、ドメインモデルと計算ロジックが簡素化されUXも向上する。

## 影響

既存のドメインオブジェクトとテーブルに以下の修正が必要:

- `IndividualValues`（個体値）/ `EffortValues`（努力値）を廃止し、`StatPoints`（能力ポイント）に置換
- `Nature`（性格）を `StatAlignment`（能力補正）に名称変更
- `StatsCalculator` の計算式を全面書き換え（新計算式を使用、`Level` 引数を廃止）
- `DamageCalculator` の `attackerLevel` パラメータを廃止しLv.50固定化
- `Individual` / `IndividualEntity` のプロパティ変更（個体値6列+努力値6列を削除、能力ポイント6列を追加、`NatureId` → `StatAlignmentId`）
- DBテーブル変更（`Natures` → `StatAlignments`、`Individuals` テーブルのカラム変更）
- 関連テスト全面更新
- Champions専用化の判断については別ADRで決定する

## 補足

### 計算式

過去の形式(HP): (種族値×2+個体値+努力値÷4)×レベル÷100+レベル+10
過去の形式(それ以外): {(種族値×2+個体値+努力値÷4)×レベル÷100+5}×せいかく補正
今回の形式(HP): 種族値+75+能力ポイント
今回の形式(それ以外): (種族値+20+能力ポイント)×能力補正

### 能力ポイントの制約

- 各ステータスの上限: 32
- 合計上限: 66
- 能力補正（Stat Alignment）: 従来の性格と同じ25種。上昇補正1.1倍、下降補正0.9倍、補正なし1.0倍
- 能力補正後の丸め: 小数点以下切り捨て
