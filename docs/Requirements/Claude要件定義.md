# **ポケモンバトルアシスタント 要件定義書**

## **1\. 概要**

### **1.1 目的**

ポケモン対戦（ランクマッチ等）を補助するWebアプリケーション。ダメージ計算・個体管理・パーティ管理・対戦中のメモ・戦績管理を統合的に提供する。

### **1.2 前提条件**

| 項目 | 内容 |
| ----- | ----- |
| 対象世代 | ポケモンスカーレット・バイオレット（Generation 9） |
| 対戦形式 | シングルバトル前提（ダメージ計算のみダブル対応） |
| テラスタル | 対応必須 |
| 対象ユーザー | 個人利用（認証なし） |
| 対象デバイス | PC（モバイル対応は拡張候補） |

### **1.3 技術スタック**

| 項目 | 技術 |
| ----- | ----- |
| フロントエンド | Blazor Web App（SSR中心） |
| バックエンドAPI | ASP.NET Core Web API（別プロセス） |
| ORM | Entity Framework Core |
| データベース | PostgreSQL（ユーザー管理サーバー上） |
| オーケストレーション | .NET Aspire |
| 外部データ | PokeAPI |
| 言語 | C\# / .NET |

### **1.4 アーキテクチャ概要**

* Blazor Web App（SSR中心）とASP.NET Core Web APIを別プロセスとして分離する  
* Blazor AppはWeb APIを介してデータにアクセスする  
* .NET Aspireでオーケストレーションする  
* PostgreSQLの接続文字列はユーザーが設定する

---

## **2\. マスターデータ管理**

### **2.1 データソース**

PokeAPIからデータを取得し、オリジナルのデータ構造に変換してPostgreSQLに保存する。

### **2.2 取得対象データ**

| データ種別 | 備考 |
| ----- | ----- |
| ポケモン種族データ | 種族名（日本語）・種族値・タイプ・重さ等 |
| 技データ | 技名（日本語）・タイプ・威力・命中・PP・分類・追加効果等 |
| 特性データ | 特性名（日本語）・効果テキスト |
| アイテムデータ | アイテム名（日本語）・効果（ダメージ計算に影響するもの） |
| タイプ相性 | 18タイプの相性表 |

### **2.3 世代とフォルムの扱い**

* 種族値・技データ等はSV（Gen 9）の値に限定する  
* フォルムチェンジ・リージョンフォーム等は**すべて内部上別の種族IDを振る**（別種族扱い）  
* 種族・技・特性等にはPokeAPI由来のID（または内部ID）を記録し、将来の多言語対応に備える

### **2.4 データ更新**

* UIの更新ボタンによる手動更新（初期実装）  
* 定期実行は拡張候補  
* PokeAPIからの取得 → オリジナル構造への変換 → PostgreSQL保存

### **2.5 言語**

* 日本語のみ（多言語対応は不要）  
* ただし内部IDを記録しておくことで将来の多言語対応を可能にする

---

## **3\. 個体管理**

### **3.1 概要**

対戦で使用するポケモンの個体情報を登録・管理する機能。登録した個体はパーティ管理・バトルモードで参照される。

### **3.2 登録項目**

| 項目 | 必須 | 備考 |
| ----- | ----- | ----- |
| 種族 | ○ | マスターデータから選択 |
| ニックネーム（NN） | × | 任意入力 |
| 性格 | ○ | 補正対象ステータスの選択 |
| 特性 | ○ | 種族に応じた選択肢 |
| 個体値（IV） | ○ | 各ステータス 0〜31 |
| 努力値（EV） | ○ | 各ステータス 0〜252、合計510以内 |
| 技 | ○ | 1〜4つ、マスターデータから選択 |
| 持ち物 | × | マスターデータから選択 |
| テラスタイプ | ○ | 18タイプから選択 |
| メモ | × | 自由テキスト |

### **3.3 機能要件**

* **Lv.50実数値の自動計算**: 種族値・個体値・努力値・性格から各ステータスの実数値をリアルタイムで計算・表示する  
* **同一種族の複数登録**: 可能（例: AS型ガブリアスとHB型ガブリアスを別個体として登録）  
* **削除**: パーティに所属している個体も削除可能。ただし**警告を表示**してから実行する  
* **インポート・エクスポート**: 後述（§8）  
* 実数値は入力変更時にリアルタイム更新される

---

## **4\. ダメージ計算プリセット個体**

### **4.1 概要**

ダメージ計算で使用した個体を、手持ち個体とは**別カテゴリ・別テーブル**で保存する機能。よく見る相手の調整パターン（例: HB特化カバルドン）を登録しておき、素早く呼び出す用途を想定。

### **4.2 管理**

* ダメージ計算画面内で管理する（詳細な管理UIは未定）  
* 登録項目は個体管理と同等（§3.2）  
* メモ欄あり  
* ダメージ計算時に使用した個体を**プリセットとして保存**する機能  
* ダメージ計算時に使用した個体を**手持ち個体としてその場で登録**する機能

---

## **5\. パーティ管理**

### **5.1 概要**

個体管理で登録した個体を6体まで選択し、1つのパーティ（PT）として登録・管理する機能。

### **5.2 機能要件**

* 個体管理で登録済みの個体を選択する（**参照、コピーではない**）  
* 最大6体  
* **順番を記録する**（先頭が1番目）  
* メモ欄あり  
* インポート・エクスポート: 後述（§8）  
* 参照先の個体が削除された場合、パーティから自動的に除外される（または欠番として扱う）

---

## **6\. ダメージ計算機**

### **6.1 概要**

ポケモンSVの対戦における正確なダメージ計算を行う機能。原作準拠の計算式を用い、各種修正値を網羅する。

### **6.2 入力項目**

#### **攻撃側・防御側共通**

| 項目 | 入力方法 |
| ----- | ----- |
| ポケモン | 自由入力 or 登録個体選択 or プリセット個体選択 |
| ステータス | 種族値・IV・EV・性格から自動計算、または実数値直接入力 |
| 特性 | 選択 |
| 持ち物 | 選択 |
| テラスタル状態 | ON/OFF \+ テラスタイプ |
| ランク補正 | \-6〜+6 |
| 状態異常 | やけど等 |

#### **技**

| 項目 | 備考 |
| ----- | ----- |
| 技選択 | マスターデータから選択 |
| 威力の手動修正 | 体重依存技等の未対応技への暫定対応 |

#### **場の状態（カードごとに個別設定）**

| 項目 | 選択肢 |
| ----- | ----- |
| 天候 | なし / 晴れ / 雨 / 砂嵐 / 雪 |
| フィールド | なし / エレキ / グラス / サイコ / ミスト |
| 壁 | リフレクター / ひかりのかべ / オーロラベール |
| バトル形式 | シングル / ダブル |

### **6.3 計算仕様**

#### **基本方針**

* **原作（ポケモンSV）準拠の計算式**を使用する  
* すべての修正値（天候・フィールド・特性・持ち物・テラスタル・壁・やけど・急所・ランク補正等）を考慮する

#### **対応する特殊ケース**

| ケース | 対応 |
| ----- | ----- |
| テラスタルによるタイプ一致補正 | ○ |
| 特性によるダメージ補正 | ○（全特性） |
| 持ち物によるダメージ補正 | ○ |
| 連続技 | ○（各ヒットの乱数を個別計算） |
| 固定ダメージ技 | ○ |
| 体重依存技 | 後回し（威力手動修正で暫定対応） |
| ダブルバトル補正 | ○（バトル形式でダブル選択時） |
| ステルスロック等の定数ダメージ | ○（確定数計算に含める） |
| 天候ダメージ（砂嵐等） | ○（確定数計算に含める） |

### **6.4 計算結果の表示**

| 表示項目 | 説明 |
| ----- | ----- |
| ダメージ範囲 | 最小〜最大ダメージ |
| HP割合 | 相手の残HPに対するダメージの割合（%） |
| 乱数16段階 | 乱数0〜15の各ダメージ値を表示 |
| 確定数 | 何発で倒せるか（ステルスロック・天候ダメージも考慮） |
| 急所時 | 急所が出た場合のダメージも併記 |

### **6.5 計算カード**

* ダメージ計算は**カード**単位で管理する  
* **複数カードを並行表示**できる  
* 既存カードの**複製**が可能  
* 場の状態（天候・フィールド・壁等）は**カードごとに独立**  
* 画面遷移時にカードは**破棄**される

---

## **7\. バトルモード**

### **7.1 概要**

実際の対戦を補助するモード。見せ合い→対戦中→終了のフローで進行し、メモ・ダメージ計算・戦績記録を統合的に提供する。

### **7.2 フロー**

PT選択 → 見せ合いフェーズ → 対戦中フェーズ → 終了（勝敗入力）

* 同一ページ内で遷移する  
* セッションの途中保存は不要  
* **戦績はバトルモード遷移時から自動保存を開始し、更新されるたびに保存される**

### **7.3 PT選択**

* 登録済みパーティを選択してバトルモードに入る

### **7.4 見せ合いフェーズ**

| 機能 | 説明 |
| ----- | ----- |
| 相手ポケモン入力 | 種族をマスターデータから選択（最大6体） |
| 自分の選出決定 | 登録PT内から選出する個体と順番を選択 |
| メモ | 相手のPT・自分の選出に関する自由メモ |

### **7.5 対戦中フェーズ**

| 機能 | 説明 |
| ----- | ----- |
| 相手の選出マーク | 相手のポケモンが登場したら手動でマーク |
| 相手の情報追記 | 判明した技・持ち物・特性・テラスタイプを逐次追記 |
| ダメージ計算カード | §6のダメージ計算機を使用 |
| ダメージ計算の制約 | 自分側: 登録PT個体から選択 / 相手側: 見せ合い時の種族から選択 |
| 相手の仮定値 | 努力値・性格等は仮定として入力。入力値は対戦中一時的に保持される |

### **7.6 終了**

* 勝敗を入力して対戦を終了する  
* 戦績に最終状態が保存される

---

## **8\. インポート・エクスポート**

### **8.1 概要**

個体・パーティのデータをテキスト形式で入出力する機能。

### **8.2 方針**

| 項目 | 方針 |
| ----- | ----- |
| フォーマット | オリジナルテキスト形式（詳細はTBD） |
| 優先事項 | 人間可読性を優先（記事への転用を想定） |
| 出力方法 | クリップボードへのコピー or テキストファイルダウンロード |
| 複数まとめ | 複数個体・複数PTをまとめてインポート・エクスポート可能 |

### **8.3 対象**

* 個体データ  
* パーティデータ

---

## **9\. 戦績管理**

### **9.1 概要**

バトルモードの対戦結果を記録・集計する機能。

### **9.2 記録項目**

| 項目 | 説明 |
| ----- | ----- |
| 日時 | 対戦日時（自動記録） |
| 勝敗 | 勝ち / 負け |
| 使用PT | バトル開始時点のPT構成の**スナップショット**（コピー） |
| 自分の選出 | 選出個体と順番 |
| 相手の選出 | 判明した相手の選出と順番 |
| 相手のPT | 見せ合い時にメモした相手のポケモン |

### **9.3 スナップショット**

* 戦績に紐づくPTデータはバトル開始時点の**コピー**  
* 後から個体やPTを編集しても過去の戦績データには影響しない

### **9.4 シーズン**

* ユーザーが手動で「統計区間（シーズン）」を作成する  
* 詳細な仕様はTBD（後回し）

### **9.5 集計機能**

| 集計項目 | 説明 |
| ----- | ----- |
| PT勝率 | パーティごとの勝率 |
| 選出率 | 個体ごとの選出率 |
| シーズン統計 | シーズンごとの上記集計 |
| その他 | TBD |

### **9.6 自動保存**

* バトルモード遷移時から保存を開始する  
* 対戦中に情報が更新されるたびに保存される  
* バトルモード終了時に勝敗が確定し、最終保存される

---

## **10\. 画面構成**

### **10.1 ナビゲーション**

トップレベルのメニューとして以下を配置する。

1. **個体管理** – 個体の一覧・登録・編集・削除  
2. **パーティ管理** – パーティの一覧・登録・編集・削除  
3. **ダメージ計算** – スタンドアロンのダメージ計算機（プリセット個体管理を含む）  
4. **バトルモード** – PT選択 → 見せ合い → 対戦中 → 終了（同一ページ内遷移）  
5. **戦績** – 対戦記録の一覧・集計

### **10.2 PC前提**

* デスクトップブラウザでの使用を前提とした画面設計  
* モバイル対応は拡張候補

---

## **11\. 拡張候補（初期スコープ外）**

以下は初期実装のスコープ外とし、将来の拡張候補として記録する。

| 候補 | 備考 |
| ----- | ----- |
| レギュレーション対応 | ポケモン・持ち物の使用可否管理 |
| 実数値逆算補助 | 確定数や耐久調整のための実数値逆算 |
| 素早さ比較支援 | 素早さ実数値の比較・一覧 |
| モバイル対応 | レスポンシブデザイン |
| クラウド対応 | サーバーレス構成・クラウドDB |
| HP・状態トラッキング | バトルモード中のHP残量・状態異常の追跡 |
| マスターデータ定期自動更新 | 定期実行によるPokeAPIデータ更新 |
| 体重依存技の完全対応 | 現状は威力手動修正で暫定対応 |
